ngx-dynamic-table-component Demo

The ngx-dynamic-table-component is a versatile Angular component that allows you to dynamically generate tables using Bootstrap. You can provide data in various formats to create tables with customizable headers, footers, and cell styles.

Using string[][]

You can provide a string[][] array to generate a simple table. For example: 

{{ examplesStrings.string }}

Using TableOptions

For more advanced configurations, you can provide a TableOptions object. This allows you to customize headers, footers, and individual cell styles, and enable sorting on specific columns. For example:

Features of TableOptions

TableOptions has the properties header, body, footer, and class. The first three are arrays or double arrays of CellOptions, which define properties to modify the styles of the cells.

Property header

The header property of TableOptions is an array of CellHeaderOptions. These options allow you to customize the style of the table header cells. Below are the available properties:

  • text: The text to be displayed in the cell.
  • bold: If the text should be bold.
  • italic: If the text should be italic.
  • class: A custom CSS class to apply specific styles to the cell.
  • colSpam: Number of columns the cell spans.
  • rowSpam: Number of rows the cell spans.
  • sortable: If the column is sortable.

{{ examplesStrings.header }}

Property body

The body property of TableOptions is a double array of CellOptions. This allows you to define the content and style of each cell in the table's body. Below are the available properties for each cell:

  • text: The text to be displayed in the cell.
  • bold: If the text should be bold.
  • italic: If the text should be italic.
  • class: A custom CSS class to apply specific styles to the cell.
  • colSpam: Number of columns the cell spans.
  • rowSpam: Number of rows the cell spans.

Example usage:

{{ examplesStrings.body }}

Property footer

The footer property of TableOptions is a double array of CellOptions, similar to the body property. However, the footer is rendered in the tfoot element and is not affected by column sorting. This allows you to display summary information or totals that remain static regardless of how the table is sorted.

  • text: The text to be displayed in the cell.
  • bold: If the text should be bold.
  • italic: If the text should be italic.
  • class: A custom CSS class to apply specific styles to the cell.
  • colSpam: Number of columns the cell spans.
  • rowSpam: Number of rows the cell spans.

{{ examplesStrings.footer }}

Property class

The class property of TableOptions allows you to apply custom CSS classes to the table. This enables you to style the table according to your specific design requirements. You can use any Bootstrap classes or your own custom classes to achieve the desired look and feel.
